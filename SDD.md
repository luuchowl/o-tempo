# o-tempo — Software Design Document

## 1. Overview

**o-tempo** is a weather data visualization tool that fetches historical meteorological data, stores it in CSV, and renders it as an interactive 3D chart using Three.js + WebGL.

Future scope: serve as a composable data layer for web applications running prediction models for business intelligence.

---

## 4. Data Pipeline (Python)

### Source API
Open-Meteo Historical Archive — free, no API key required.  
Endpoint: `https://archive-api.open-meteo.com/v1/archive`

### Date Range
November 1, 2025 → June 8, 2026 (8 calendar months)

### Sampled Hours
`0:00, 3:00, 6:00, 9:00, 12:00, 15:00, 18:00, 21:00`  
(8 samples/day → aligns with the 8 Y-axis slots in the chart)

### CSV Schema
```
date_time;latitude;longitude;temperatura;sensacao_termica;umidade_relativa;chuva_mm
```

| Field              | Source variable            | Unit  |
|--------------------|----------------------------|-------|
| `date_time`        | ISO-8601 datetime string   | —     |
| `latitude`         | fixed                      | °     |
| `longitude`        | fixed                      | °     |
| `temperatura`      | `temperature_2m`           | °C    |
| `sensacao_termica` | `apparent_temperature`     | °C    |
| `umidade_relativa` | `relative_humidity_2m`     | %     |
| `chuva_mm`         | `precipitation`            | mm    |

Output file: `data/weather.csv`

## 5. Frontend — Three.js 3D Chart

### Chart Concept
A 3D bar chart rendered as a grid of cubes. The user can orbit the chart in **yaw** (rotate around the vertical axis) to inspect depth.

### Grid Dimensions
| Axis | Meaning                          | Values (8 slots)                          |
|------|----------------------------------|-------------------------------------------|
| X    | Month                            | Nov, Dec, Jan, Feb, Mar, Apr, May, Jun    |
| Y    | Time of day                      | 0:00, 3:00, 6:00, 9:00, 12:00, 15:00, 18:00, 21:00 |
| Z    | Average temperature for the cell | derived from CSV aggregation              |

Each cell `(month, hour)` holds the **mean temperature** for all readings that fall in that month × hour bucket.

### Cube Geometry
- Unit size: 1 × 1 × 1 meter base
- Pivot at **ground level** (Y-bottom of cube = 0)
- Height scaled to temperature value
- Grid spacing: 1 unit (no gap)
- Total grid footprint: 8 × 8 units

### Camera
- **Orthographic projection**
- Frustum shows ±6 units on both X and Y (12-unit view width/height)
- Initial rotation: 45° around X axis (isometric-style view)
- User interaction: **yaw orbit only** (rotate around the world Y axis)

### Color Mapping
Cube color interpolates linearly from `cold color → hot color` based on normalized Z (temperature).

| Temperature | Color (suggestion) |
|-------------|-------------------|
| Min (≈ 14°C) | `#4fc3f7` (cool blue) |
| Max (≈ 34°C) | `#ff5722` (hot orange-red) |

### Axes / Labels
- X labels: month abbreviations (Nov … Jun) rendered as sprites below the grid
- Y labels: hour strings (0h … 21h) rendered as sprites to the side
- Z: no label on bars; a color legend is shown in the HUD
